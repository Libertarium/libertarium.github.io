Акт по проверке выполнения п. 23 особых условий лицензии N5769 на предоставление услуг телематических служб ЗАО "БСК"

23.11.1998

АКТ

по проверке выполнения пункта " 23 особых условий лицензии " 5769
на предоставление услуг телематических служб ЗАО "Байярд-Славия Коммуникэйшнс"

"23" ноября 1998г. г.Волгоград

Государственными инспекторами по надзору за связью Управления Госсвязьнадзора по Волгоградской области Костяной Л.Г., Серебряковым С.В. в присутствии генерального директора ЗАО "Байярд-Славия Коммуникэйшнс" 23.11.1998г. проведена проверка выполнения пункта " 23 особых условий лицензии " 5769 на предоставление услуг телематических служб и установлено:

При обследовании объекта связи установлено, что объект принадлежит ЗАО "Байярд Славия Коммуникэйшнс", расположен по адресу: г.Волгоград, ул. Репина, 100.
Руководитель предприятия -- генеральный директор ЗАО "Байярд Славия Коммуникэйшнс" -- Сыров О.П.
Лицензия " 5769 выдана 24.01.97г. срок действия до 24.01.2002г.
Проверено выполнение пункта лицензии "23.

1. Структура сети

В г.Волгограде сеть ЗАО "Байярд-Славия Комуникэйшнс" представлена опорной сетью, состоящей из узла передачи данных для предоставления услуг, предусмотренных протоколами TCP/IP. Для организации межузловой связи (между узлом ЗАО "Баяйрд-Славия Коммуникэйшнс" и узлом поставщика услуг Internet используется спутниковый канал.

Доступ абонентов к сети ЗАО "Байярд-Славия Коммуникэйшнс" осуществляется по коммутируемым каналам сети ТФОП.

2. Виды используемой аппаратуры связи:

Табл. 1.

Наименование Номер модели Количество, шт. " сертификата
1. Модем USR Courier V.Everething 24 ОС/1-ТФ-86
2. Мультиплексор DigiBoard 16-Е 1 Не треб.
3. Маршрутизатор IBM 2210 Е-12 1 Не треб.
4. IP-сервер Р5 2 Не треб.

Используемое программное обеспечение: Windows NT 4.0 Server, Microsoft Proxy Server 2.0, NT Mail 3.03.

На представленное программное обеспечение имеются лицензионные соглашения.

3. Доступ к сети ЗАО "Байярд-Славия Комуникэйшнс" на территории г.Волгограда осуществляется через сеть ТФОП ВОАО "Электросвязь", лицензия " 3019.

4. Вьшолнение особых условий лицензии

По п.23: В результате проведенной проверки установлено, что при эксплуатации телематических служб лицензиата на узле передачи данных ЗАО "Байярд-Славия Коммуникэйшнс" при помощи аппаратно-программвых средств реализована возможность контроля за сообщениями электронной почты и отслеживания маршрута перемещения пользователя по сети Internet, что является свидетельством наличия возможностей лицензиата в оказании содействия судебно-следственным органам.

Также лицензиатом представлены копии исходящих писем в УФСБ по Волгоградской области:

1) от 28.10.1997г. с просьбой о разработке плана мероприятий по СОРМ для узла передачи данных ЗАО "Байярд-Славия Коммуникэйшнс";
2) от 27.10.1998г., в котором сообщается о возможности сотрудничества администрации узла ЗАО "Байярд-Славия Коммуникэйпшс" с УФСБ по Волгоградской области.

Выводы: возможность лицензиата в оказании содействия судебно-следственным органам имеется.

Лицензиат согласен и готов выполнять п.23. лицензии " 5769 согласно Федерального закона об ОРД " 144-03 от 12.08.95г., утвержденного правительством РФ.

Прилагается:

  1. Протокол проверки возможности проведения оперативно-розыскных мероприятий на 2-х листах.
  2. Копия письма от 28.10.1997г. в УФСБ по Волгоградской области.
  3. Копия письма от 27.10.1998г. в УФСБ по Волгоградской области.

Протокол проверки возможности проведения оперативно-розыскных
мероприятий на узле передачи данных ЗАО "Байярд-Славия Коммуникэйшнс".

1. Проверка возможности контроля сообщений электронной почты:

1) Для проведения проверки заводится три учетные записи пользователей почтового сервера mail.bayard.ru с именами пользователей tester1, tester2, tester3. Предполагается, что пользователь tester3 -- орган, осуществляющий оперативно-розыскную деятельность, tester2 -- лицо, в отношении которого осуществляется оперативно-розыскная деятельность, tester1 -- респондент.
2) Оператор узла, пользуясь полномочиями администратора, регистрирует пользователя tester3 с полномочиями органа, осуществляющего оперативно-розыскную деятельность, т.е. включает полное отображение содержимого почтового ящика пользователя tester2 на почтовый ящик пользователя tester3.
3) В MsOutlook заводится три учетных записи пользователей с именами tester1, tester2, tester3.
4) Пользователем tester1 создается проверочное сообщение и отправляется пользователю tester2.
5) Пользователем tester2 принимается отправленное проверочное сообщение.
6) Пользователем tester3 также принимается проверочное сообщение. Содержания сообщений, полученных пользователями tester3 и tester2 идентичны (в поле From: (отправитель) конверта электронного письма, полученного пользователем tester3 указан пользователь tester1, а в поле То: (получатель) указан пользователь tester2).

Аналогичная проверка проведена для случая, если отправитель электронного письма не является пользователем почтового сервера mail.bayard.ru, т.е. письмо отправляется с какого-либо доступного почтового сервера Internet, например: www.usa.net.:

1) Пользователь tester1, регистрируется на узле www.usa.net.
2) Пользователем tester1 создается проверочное сообщение и отправляется пользователю tester2.
3) Пользователем tester2 принимается отправленное проверочное сообщение.
4) Пользователем tester3 также принимается проверочное сообщение. Содержания сообщений, полученных пользователями tester3 и tester2 идентичны (в поле From: (отправитель) электронного письма, полученного пользователем tester3 указан пользователь tester1@usa.net, а в поле То: (получатель) указан пользователь tester2@bayard.ru).

2. Контроль за маршрутом перемещения пользователя по сети Internet.

Программными средствами извлекается информация о маршруте перемещения пользователя по сети Internet с точностью до каждого принятого пользователем файла. Для этого формируются файлы отчетов: condis.log, w3981123.log.

1) condis.log -- содержит информацию о соответствии присвоенного сервером DHCP IP-адреса пользователя имени пользователя с указанием начала и конца сеанса. Таким образом по имеющемуся IP-адресу можно однозначно аутентифицировать имя пользователя в указанный момент времени;
2) w3981123.log -- содержит IP-адрес, время, URL (Uniform Resource Locator), в котором указывается протокол запроса, местоположение запрашиваемого ресурса (хост, на котором расположен ресурс), и наконец, имя запрашиваемого ресурса.
