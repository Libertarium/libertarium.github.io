Технические требования к каналам

обмена информацией между СОРМ и ПУ

для сетей подвижной радиосвязи.

1. Интерфейс связи между СОРМ и ПУ

1.1. Связь между СОРМ и ПУ должна осуществляться по каналам передачи данных для передачи управляющей информации и информации о наблюдаемых соединениях и по контрольным соединительным линиям (КСЛ) для трансляции информации, передаваемой в контролируемых разговорных трактах.

1.2. Должны быть обеспечены два варианта организации связи между СОРМ и ПУ. Выбор варианта согласовывается с заказчиком.

1.3. При первом варианте связь между СОРМ и ПУ должна осуществляться с помощью цифровой системы передачи.

1.3.1. Цифровая система передачи должна быть образована первичной группой ИКМ-30 со скоростью 2,048 Мбит/с в соответствии с Рек.G.732 МСЭ-Т с канальными интервалами 64 Кбит/сек. Количество первичных групп ИКМ-30 должно соответствовать данным таблицы 1 Дополнения 1.

1.3.2. Канальные интервалы 1..15, 17..29 должны использоваться для трансляции информации, передаваемой в контролируемых разговорных трактах, а также для передачи идентификатора контролируемого вызова многочастотным способом.

1.3.3. Канальный интервал 16 может использоваться для линейной сигнализации управления оборудованием ПУ.

Примечание: для передачи линейных сигналов и идентификатора может использоваться типовая система межстанционной сигнализации.

1.3.4. Должна быть предусмотрена возможность связи между СОРМ и ПУ без передачи линейных сигналов и (или) идентификатора.

1.3.5. Канальные интервалы 30, 31 должны образовывать каналы передачи данных с помощью модемов для обмена информацией между СОРМ и ПУ.

1.3.6. Канальный интервал 30 должен использоваться для передачи управляющей информации (канал 1). Канальный интервал 31 должен использоваться для передачи информации о наблюдаемых соединениях (канал 2).

Примечания:

1. Допускается перераспределение канальных интервалов для образования контрольных соединительных линий и каналов передачи данных.

2. Для увеличения пропускной способности допускается использование нескольких каналов передачи данных.

1.4. При втором варианте связь между СОРМ и ПУ должна осуществляться с помощью цифровой системы передачи в соответствии с п.1.3.1. и по физическим парам телефонного кабеля.

1.4.1. Канальные интервалы с 1 по 29 должны использоваться в соответствии с требованиями п.п.1.3.1-1.3.4.

1.4.2. Каналы передачи данных (канал 1 и канал 2) должны организовываться по физическим парам телефонного кабеля с диаметром жил 0,7 мм на расстояние до 16 км.

1.5. Обмен информацией по каналам передачи данных должен осуществляться с помощью модемов с 2-х или 4-х проводным окончанием.

1.5.1. Обмен информацией должен осуществляться в дуплексном режиме со скоростью, обеспечивающей минимальное время передачи сообщений.

1.5.2. Канал передачи данных (КПД) должен обеспечивать коэффициент ошибок по битам не более 10-5 при коэффициенте ошибок по битам в линии связи не более 10-3 при воздействии белого шума в полосе частот 0,3-3,4 КГц и соотношении сигнала/шум плюс 12 дБ.

1.5.3. Параметры и тип модема должны соответствовать рекомендациям МСЭ-Т на аппаратуру передачи данных.

1.6. Должен быть обеспечен контроль исправности каналов обмена информацией между СОРМ и ПУ. При повреждении аппаратуры каналов обмена информацией или кабеля между ПУ и станцией должна быть прекращена передача данных на ПУ. При этом данные утрачиваются.

1.7. Количество разговорных трактов и каналов передачи данных проектируется таким образом, что при выходе из строя ИКМ-30 или канала передачи данных резервной емкости будет достаточно как для передачи данных, так и речевых сигналов.

2. Протокол обмена информацией в канале передачи данных

2.1. Протокол обмена данными между СОРМ и ПУ должен соответствовать рекомендации Х.25 МСЭ-Т.

2.2. Протокол Х.25 должен включать в себя сетевой, канальный и физический уровни. В качестве физического уровня должен использоваться интерфейс V.24 МСЭ-Т.

2.3. Средства реализации протокола связи в КПД должны обеспечивать установку переменных параметров второго и третьего уровней в соответствии с рекомендациями Х.25 МСЭ-Т.

2.4. Значение переменных параметров могут уточняться на этапах разработки рабочей документации на СОРМ и опытной эксплуатации.

2.5. Для определения работоспособности каналов передачи данных (ПД) с протоколом обмена Х.25 на уровне пользователя должны использоваться "сигналы жизни", передаваемые между оборудованием каналов ПД СОРМ и ПУ в виде пробного пакета Х.25.
