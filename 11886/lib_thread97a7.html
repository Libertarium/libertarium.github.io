    Здравствуйте, Анатолий!

    Что сказать - крутой notebook. Эта крутизна вполне позволяет установить на этот компьютер средства Dial-Up сервера и использовать его в качестве мобильного Сервис-Провайдера. Например, Вы делаете accomodation в гостиничном номере, или на квартире, сообщаете номер телефона своим друзьям, подключаете свой сиреневый аппарат к телефонной линии через модем, идете в My Computer\Dial-Up Networking\Connections\Dial-Up Server и выставляете опцию Allow caller access, с паролем или без оного.

    Ваши друзья соединяются с Вашим компьютером практически таким же образом, как и со своими Интернет Сервис-Провайдерами, а какой сервис вы им предoставите и позволите - дело хозяйское, вплоть до E-mail, WWW and FTP, не говоря уже о традиционных File and Print Services. Достоинства такой схемы - ФСБ не скоро доберется со своим СОРМ-2 до Вашего сиреневого монстра.

    В принципе, такую схему можно применять к любой паре компьютеров, имеющих выход в телефонную сеть через модем, поэтому начну описание с прожиточного минимума - для Win95-х компьютеров.

    1. Для начала требуется сходить на www.microsoft.com/windows95/downloads/, раздел NETWORKING, пунkты:

    -Microsoft Virtual Private Networks
    -Dial-Up Networking 1.3
    -MS DUN 1.3 and Winsock 2 year 2000 update

    и забрать оттуда три файла:

    -vpnupd.exe
    -msdun13.exe
    -y2kvdhcp.exe

    Еще потребуется исходный CD-ROM диск с Win95.

    2. После инсталляции всех трех программ требуется проверить,что программные средства сервера (Dial-Up Server) действительно установлены. Для этого - сходить Control Panel\Add Remove Programms\Windows Setup\Communications\Details\Components.

    В списке должно быть отмечено "птичкой" строка "Dial-Up Server". Если это не так, то отметить - и на "OK" для установки.

    3. Проверить, что на компьютере действительно появилась возможность сконфигурировать Dial-Up Server:

    My Computer\Dial-Up Networking\Connections и среди прочих опций (Connect, Make New connection, Settings) должна появиться строчка "Dial-Up Server", и если расkрыть ее, то должны появиться опции настройки:

    -Allow caller access
    -Password Protection
    -Server Type
    -Advanced Options

    В Server Type требуется выставить:

    PPP:Internet, Windows NT Server, Windows 95

    В Advanced Options необходимо отметить птичками:
    -Enable Software Compression
    -Require Encrypted Password

    4. Эта группа действий практически мало отличается от тех, что выполняются при предоставлении File and Printer Sharing на локальной сети в составе рабочей группы (или Pear-to-Pear connections):

    4.1. Установить Client for Microsoft networks (Control Panel\Network\Add\Client...)

    4.2. Конфигурировать свою Network по протокольной части:
    Control Panel\Network\Configuration\Add\Protocol\Add\Microsoft

    В списке Network Protocols для PPP подходят:

    IPX/SPX-compatible
    NetBEUI
    TCP/IP

    Самый быстрый (и простой-не надо ничего конфигурировать) из них - NetBEUI, им можно и ограничиться (NetBEUI отлично работает в LAN до 200 компьютеров).

    Если на Вашем компьютере использовать TCP/IP для Интернета и NetBEUI для Dial-Up Server'a, то на компьютере Ваш Dial-Up Adapter будет связан с двумя протоколами - TCP/IP and NetBEUI. Обязательно проверьте, что в связке TCP/IP->Dial-Up Adapter НЕ ВЗВЕДЕНО "I want to enable NetBIOS over TCP/IP" (Control Panel\Network\Confogurations\TCP/IP-->Dial-Up Adapter\Properties\NetBIOS) просто для того, чтобы из Интернета не были видны NetBIOS-ресурсы вашего компьютера.

    4.3. Активизировать Sharing (Conrol Panel\Network\File and print sharing) и сделать разделяемым что считаете нужным. Через Property обьектов (disk, folder, subfolder, printer) можно сделать их разделяемыми ресурсами с различными permissions и с паролями и без.

    4.4. Установить Computer Name and Workgroup (Control Panel\Network\identification tab). Понятно, что имена компьютера-клиента и компьютера-сервера должны быть любыми до 15 символов длиной, а принадлежать они должны к одной рабочей группе.

    4.5. Установить Share-level access control (Control panel\Network box\Access Control tab)

    5. Этот пункт больше относится к компьютеру клиента, где требуется сконфигурировать Make New Connections. Хотя любой из компьютеров может выступать и в роли Dial-Up Сервера, и в роли Клиента.

    Порядок конфигурации практически ничем не отличается от того, что выполнялся для звонков к Вашему Интернет Сервис-провайдеру. В конечном итоге, Property вашего соединения должны выглядеть так:

    General Tab: - phone number куда "звонить"
    - connect using -тот же модем

    Server Types:

    Type of server:PPP:Internet, Windows NT server,Windows 95

    Advanced options:
    ^Log on to network
    ^Enable software compression
    ^Require encripted password
    ^Require data encription

    Allowed network protocol:
    ^NetBEUI
    IPX/SPX
    TCP/IP
    Понятно, что по согласованию сторон можно использовать любые, хоть все сразу, протоколы.Единственно, что при выборе TCP/IP желательно пользоваться private IP addresses for your internal network, ranges:

    10.0.0.0 - 10.255.255.255
    172.16.0.0 - 172.31.255.255
    192.168.0.0 - 192.168.255.255

    Как будто бы - все...
    Понятно - недостаток этой схемы в том, что если кто-то один уже соединился с вашим сервером, то для остальных он уже не доступен по причине занятости линии. Выход - перейти на Virtual Private Networks соединения, Интернет не занят никогда...
